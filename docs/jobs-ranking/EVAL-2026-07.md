## 2. What works (evidence-backed)

1. **The LLM reranker is the single biggest quality lever.** P@5 0.8 vs 0.2 for both baselines. It ejected all four baseline top-10 mismatches (268139 .NET LATAM, 265622 $300 gig, 267087 Africa SRE, 265629 clinical AI) from the top 20 entirely and promoted BitPay 7→1, Shepherd 13→2, This Dot 12→3, SerpApi 14→8 — all judged correct. Zero gold-WORST in any reranked list (rerank_90 + all 3 prompt runs).
2. **The `remote=Full` filter predicate is semantically correct.** All 12 gold-BEST are `remote=Full`; the filter (matchJobs.js:91) lost none of them to its own predicate.
3. **Combined-score composition is internally consistent.** All 11 files are correctly sorted (decayed_similarity, or combined_score monotone in rerank_score with decay tiebreak), no duplicates, all HTTP 200.
4. **Feedback plumbing works as designed:** not_interested/dismissed jobs are removed and >=2 rejections perturb the embedding (matchPipeline.js:47-86) — the personalization ran without errors on all authed calls.
5. **The rerank window covers the returned page at default sizes:** at top=20, min(30, top*2)=30 > 20, so every returned row carries rerank_score (20/20 observed in all 4 rerank files).

## 3. What doesn't — failure → code cause → example

### F1. Retrieval pool is capped by RAW similarity before decay — silently drops gold
**Code:** `matchJobs.js:31` — `match_count: top * 5` fetches only the top 100 candidates (for top=20) by *raw* similarity; `timeDecayScore` is applied *after* (matchJobs.js:78-79). A fresh job whose raw sim ranks 101st can decay-score above the returned #1, but it never enters the pool.
**Evidence:** gold-BEST 267093 (Zeitlabs, corpus decayed 0.392), 268565 (Orbit, 0.392), 268138 (BW Events, 0.387) beat every 90-day file's top-20 cutoff (e.g. remote_90 #20 = 0.348) yet are absent from EVERY 90-day run. baseline_30 retrieves 267093 (smaller date window → they make the raw-sim top-100), proving retrievability. All 12 gold-BEST are `remote=Full` with corpus decayed >=0.354 — a correct remote_90 would contain the entire gold-BEST set in its top 20; it contains 4. This one bug costs 3-8 gold jobs per query regardless of filter.

### F2. Recency decay outweighs the entire semantic signal
**Code:** `vectorMath.js:56` — `0.85 * similarity + 0.15 * 0.5^(age/14d)`.
**Math:** raw sims span only ~0.36-0.44, so the weighted similarity spread is 0.85 × 0.08 ≈ **0.068**, while the recency term swings **0.15** (today) → ~0 (90d). Freshness is worth 2.2× the whole relevance band.
**Evidence:** gold-WORST 268139 (legacy .NET 4.8 WebForms, LATAM-only, ~$50k/yr, mid-level) at **#1** in baseline_90/30, remote_90, minsal_150, keyword_ai (raw rank 6, lifted by freshness). baseline_30 pushed raw-rank-20/19/18 jobs to #5/#6/#7.

### F3. min_salary filter is a no-op that fails open
**Code:** `matchJobs.js:94` — `minSalary && j.salary_usd && j.salary_usd < minSalary * 1000` skips jobs with null `salary_usd`; and the ingest parser never populates `salary_usd` (**NULL on 100/100 corpus jobs**).
**Evidence:** the "min $150k" list is headed by 268139 ($3.5-4.9k/mo), contains 265622 ($300-1500 total) at #4 and Spacelift ($80-110k, stated in text) at #8, while gold-BEST 265713 (Akkio, $160-240k) and 265690 (Mitty, $150k base) — which explicitly meet the bar — never appear.

### F4. Keyword search is substring-match over the whole JSON row
**Code:** `matchJobs.js:97-100` — `JSON.stringify(j).toLowerCase().includes(search.toLowerCase())`.
**Evidence:** `search=ai` matches 98/100 corpus jobs ("maint**ai**n", "av**ai**lable", "tr**ai**ning", "blockch**ai**n", "B**ai**du"). Its #1 is the .NET job matched via "maintain"; its #4 (265622) explicitly **forbids** using AI. 10 of its top 20 contain no standalone "AI" token. It excluded exactly one job — by luck.

### F5. Prompt-steered search barely steers retrieval
**Code:** `searches/route.js:118-142` — the HyDE doc is generated from `Candidate background + prompt` (route.js:121), and the long resume dominates the short prompt, so the "ideal posting" is nearly identical for every prompt; the final embedding then keeps 35% raw resume (`PROMPT_WEIGHT = 0.65`, route.js:12).
**Evidence:** 16-17/20 overlap between the three prompt lists; same-job similarity deltas <=0.013 (mean 0.007) across prompts. The reranker can only shuffle a fixed generic pool, so 265679 Grafana "AI/Agentic apps" never appears in the *agents* prompt results and 265716 "Founding Engineer" never appears in the *founding* prompt results.

### F6. The reranker doesn't know where the candidate lives
**Code:** `embeddings.js:33-53` (resume text: label/summary/skills/work/education/projects — **no `basics.location`**) and `resolveEmbedding.js:22-31` (search-profile resumeText is label+summary+skills only). `rerank.js:53` tells the model to "consider location/remote fit", but the candidate's Melbourne/remote-only constraint is never in the context.
**Evidence:** prompt_senior-ai-llm-agents put onsite-CA Shepherd at #2, 3-days/week-SF Weave Bio at #3, hybrid-Madrid Adyen at #4 — 6 of top 10 onsite/hybrid for a remote-only Melbourne candidate. rerank_90 gave US+EU-only Runway rerank 9 at #5.

### F7. Coarse integer rerank scores create tie-bands broken by recency, not fit
**Code:** `rerank.js:60` clamps to integers 1-10; `matchJobs.js:127-131` breaks ties by decayed similarity via the 0.35 vector term.
**Evidence:** six jobs tied at rr=9 in rerank_90; stale hybrid-NYC Campus (posted May 8, rr=9) landed #6 above gold-BEST SerpApi (rr=8) purely on tie mechanics. Silent fallback `rerank_score: 5` on any parse error (`rerank.js:63`) is indistinguishable from a real "5".

### F8. global_remote: unreliable label used as a hard filter
**Code:** `globalRemote.js:46` — any `countryCode` present ⇒ `false` unless the text says worldwide; `matchPipeline.js:121-123` then hard-filters.
**Evidence:** SerpApi (explicitly pays "local avg +20% outside US") classified `false` — wrong drop. Gold-BEST 268138 is `global_remote=true` in corpus yet missing from the global_remote_90 result (total=8 rules out a cap — the classification is cache/run-dependent). Net result: an 8-item list containing 3 gold-WORST ($300 gig #1, RLHF labeling #3, entry-level marketer #6) and 1 gold-BEST.

### F9. Non-deterministic scoring between calls
**Code:** `resolveEmbedding.js:70-76` — HyDE is regenerated (fresh LLM output → different embedding) on every GET; nothing is cached.
**Evidence:** same job/resume pair jitters up to ±0.015 across files (268139: 0.389-0.404; 265622: 0.413-0.428) — enough to reorder adjacent ranks in a 0.07-wide score band, and to shift which jobs make the raw-sim pool boundary (compounds F1).

### F10. The CLI/TUI shows a score that didn't produce the ordering
**Code:** `packages/job-search/bin/cli.js:170` prints raw `similarity`; `src/tui/JobList.js:140-141` same. The list is actually sorted by `decayed_similarity` or `combined_score`, so users see "Score 0.44" above "Score 0.40" inverted — e.g. baseline_90 #1 (sim 0.398) sits above #2 (sim 0.441).

### F11. POST /api/v1/jobs is unauthenticated with no rate limit
**Code:** `route.js:110-159` — no `authenticate()` call; every request burns an OpenAI embedding (`route.js:132`) plus a global-remote LLM classification (`matchPipeline.js:119`). Also `shouldRerank: false` hard-coded (route.js:144) and no prompt field — there is no transient prompt-search path at all.

| 1 | **Widen/re-order the retrieval pool before decay** | `matchJobs.js:31`: `match_count: Math.max(300, top * 10)` (RPC returns only id+similarity, cheap), or compute decay in SQL and order by it in `match_jobs_v5` | **Highest.** Recovers the 8/12 gold-BEST missing from every 90-day run; a correct remote_90 would then hold the entire gold set in its top 20 | S |
| 2 | **Rebalance decay so freshness can't beat relevance** | `vectorMath.js:56`: `0.95 * sim + 0.05 * recencyBoost` (caps freshness swing at 0.05 < the 0.068 similarity band); or min-max normalize sims within the candidate batch before blending | Removes gold-WORST 268139/junk from #1 across five strategies; makes baselines usable when rerank is off | S |

| 9 | **Cache embeddings for determinism** | Cache resume+HyDE embedding per (username, resume-hash) with a TTL in `resolveEmbedding.js`; reuse across requests | Eliminates ±0.015 jitter that reorders near-ties and shifts the pool boundary between refreshes | S |
| 10 | **Show the sorting score in CLI/TUI** | `cli.js:170` and `JobList.js:140-141`: display `combined_score ?? decayed_similarity` (and label the column); keep raw sim in `--json` | Users stop seeing "higher score below lower score"; trust in the ranking | S |

- corpus_100 note: with `rerank=true` at top=100 only the top min(30, top*2)=30 rows would ever be reranked (`matchJobs.js:109`) — corpus was correctly run with rerank off.
